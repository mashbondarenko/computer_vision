# HW1: Стабилизация камеры

Исходное видео: `20260309_215857.mp4`

## Что реализовано

На видео построен полный пайплайн стабилизации камеры:

1. детекция опорных точек `goodFeaturesToTrack`;
2. вычисление **разреженного оптического потока Lucas–Kanade** между соседними кадрами;
3. оценка глобального движения камеры через `estimateAffinePartial2D + RANSAC`;
4. приведение найденного преобразования к жёсткой модели `(translation + rotation)`, чтобы не накапливать ложный масштаб;
5. накопление траектории камеры;
6. сглаживание траектории скользящим окном радиуса `10` (окно `21` кадр);
7. компенсация движения через `warpAffine`;
8. подавление граничных артефактов через `BORDER_REFLECT` + центральный crop `49 px` по X и `38 px` по Y с последующим resize обратно к исходному размеру.

## Параметры

- Оценка движения ведётся на уменьшенной копии кадра: `scale=0.25`
- LK: окно `(15, 15)`, уровней пирамиды `2`
- Детектор точек: `maxCorners=220`, `qualityLevel=0.01`, `minDistance=10`
- Глобальная модель: `estimateAffinePartial2D` + RANSAC, `ransacReprojThreshold=2.5`
- Сглаживание: moving average, радиус `10`

## Характеристики обработанного ролика

- Формат кадра после декодирования OpenCV: **1080×1920**
- FPS: **29.998**
- Обработано кадров: **284**
- Длительность обработанной последовательности: **9.47 с**

## Ключевые численные результаты

- Среднее число отслеженных точек на пару кадров: **218.4**
- Медиана числа отслеженных точек: **220.0**
- Средняя доля RANSAC-inliers: **91.4%**
- Снижение std второй разности траектории:
  - X: **1.389 → 0.304** (78.1%)
  - Y: **1.681 → 0.337** (79.9%)
  - angle: **0.002464 → 0.000368** (85.1%)

Это означает, что высокочастотная дрожь заметно уменьшилась, при этом общая медленная панорама сохранилась.

## Наблюдения по качеству стабилизации

Стабилизация работает хорошо на большей части ролика, но есть и проблемные зоны:

1. **Сильный параллакс.** В кадре доминируют крупные цветы на переднем плане, а стол, книги и штора находятся на других глубинах. Одна глобальная affine/rigid-модель не может одновременно идеально совместить все плоскости.
2. **Ручное перемещение телефона вокруг сцены.** Это не чистое вращение камеры; есть заметная трансляция, из-за чего локальные потоки на ближних цветах и дальнем фоне расходятся.
3. **Пересвет на светлых лепестках и шторах.** На этих участках теряется стабильная текстура, и часть треков становится менее надёжной.
4. **Слабая текстура на однородных областях.** Светлая штора и гладкие блики дают мало хороших corner-feature.
5. **Компромисс crop / поле зрения.** Чтобы скрыть артефакты warping-а, пришлось обрезать примерно 4.5% ширины и 2.0% высоты с каждой стороны соответствующих осей.

## Почему выбран именно LK

Для этой сцены разреженный Lucas–Kanade оказался практичным выбором:

- на столе и предметах достаточно углов и текстурных деталей;
- LK быстрее, чем плотный поток, и хорошо подходит для оценки **глобального** движения камеры;
- в сочетании с RANSAC позволяет отбрасывать часть ошибочных локальных треков.
